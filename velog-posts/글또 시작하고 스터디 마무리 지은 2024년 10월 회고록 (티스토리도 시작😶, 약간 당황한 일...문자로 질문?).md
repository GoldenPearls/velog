<h4 id="1-아키텍처란">1. 아키텍처란?</h4>
<p><strong>아키텍처(Architecture)</strong>는 시스템이 어떻게 구성되고 동작하는지 큰 그림에서 보여주는 설계도입니다. 소프트웨어 아키텍처는 각 구성 요소가 어떤 역할을 수행하는지, 서로 어떤 방식으로 연결되고 데이터를 주고받는지, 시스템의 주요 흐름과 제약을 정의합니다. </p>
<blockquote>
<p>아키텍처는 소프트웨어의 안정성, 확장성, 유지보수성 등을 결정하는 중요한 요소로, 시스템의 큰 틀을 이해하고 설계 방향을 잡는 데 도움을 줍니다.</p>
</blockquote>
<h4 id="2-기존-서비스를-하나하나-분석하는-방법">2. 기존 서비스를 하나하나 분석하는 방법</h4>
<p>프로젝트를 분석하려면 다음 순서대로 진행해보는 것이 좋습니다.</p>
<p><strong>1) 전체 구조 파악</strong>
우선, 프로젝트의 전체적인 구조와 기능 흐름을 이해합니다. 이를 위해서는 다음을 살펴보세요:</p>
<ul>
<li><p>시작점: 애플리케이션의 메인 진입점을 확인합니다. 스프링에서는 일반적으로 @SpringBootApplication이나 서블릿 설정이 시작점이 됩니다.</p>
</li>
<li><p>레이어: 프로젝트의 레이어별 구성을 파악합니다. 대부분의 스프링 프로젝트는 다음과 같은 레이어로 구성됩니다.</p>
</li>
<li><p>Controller: 요청을 받아 비즈니스 로직에 전달하고, 결과를 클라이언트에 반환.</p>
</li>
<li><p>Service: 비즈니스 로직을 수행하며, 데이터 액세스나 기타 처리 로직을 구현.</p>
</li>
<li><p>Repository/DAO: 데이터베이스와의 통신을 담당하며, CRUD 작업을 수행.</p>
</li>
</ul>
<p><strong>2) 각 레이어의 역할 확인</strong>
스프링 프로젝트의 각 레이어가 어떻게 구성되고 상호작용하는지 분석합니다.</p>
<ul>
<li>Controller: 어떤 요청(URL)들이 있고, 요청을 어떻게 처리하는지 파악합니다. 각 컨트롤러는 HTTP 요청을 받아서 적절한 서비스로 전달하는 역할을 합니다.</li>
<li>Service: 비즈니스 로직을 담당하며, 비즈니스 규칙이나 다양한 연산을 처리합니다. 각 서비스 메소드가 무엇을 하는지 주석을 달아보면 이해가 쉬워집니다.</li>
<li>Repository/DAO: 데이터베이스에 접근하여 데이터를 읽고 쓰는 작업을 수행합니다. 주요 테이블이나 엔티티를 확인하고, 어떤 데이터를 관리하는지 파악합니다.</li>
</ul>
<p><strong>3) 데이터 흐름 이해</strong>
클라이언트가 요청을 보내면 컨트롤러에서 받아서 서비스로 전달하고, 서비스는 데이터베이스와의 통신을 위해 리포지토리를 호출하는 등 데이터의 흐름을 이해해야 합니다. 이를 위해:</p>
<p>주요 엔티티를 확인하고, 데이터가 어떻게 생성되고 변형되며 전달되는지 파악합니다.</p>
<blockquote>
<p><strong>DTO(Data Transfer Object)</strong>나 <strong>VO(Value Object)</strong>를 활용해 데이터가 계층 간 어떻게 이동하는지 분석합니다.</p>
</blockquote>
<p><strong>4) JSP와 뷰(View) 구성 요소 이해</strong>
JSP는 프론트엔드 뷰를 담당하며, 서버에서 동적으로 HTML을 생성하여 클라이언트에 전달합니다.</p>
<p>JSP 페이지를 통해 프론트엔드 구성과 뷰 로직을 확인합니다.
JSTL이나 EL(Expression Language)을 활용하여 동적으로 데이터를 렌더링하는 부분을 살펴보고, MVC 구조에서 JSP가 어떻게 역할을 하는지 이해합니다.
<strong>5) 의존성 분석</strong>
스프링에서는 <strong>DI(Dependency Injection)</strong>를 통해 객체 간 의존성을 관리합니다. 이로 인해 테스트와 유지보수가 용이해집니다.</p>
<p>각 클래스에 주입되는 <strong>빈(Bean)</strong>들을 확인하여, 클래스 간 의존성을 파악합니다.</p>
<blockquote>
<p>@Autowired, @Component, @Service, @Repository 등의 애노테이션을 사용해 스프링이 객체를 관리하는 방식을 이해합니다.</p>
</blockquote>
<p><strong>6) 기능별 흐름 추적</strong>
특정 기능이나 요청을 기준으로 요청-응답의 흐름을 따라가 보세요.</p>
<p>예를 들어, 로그인 기능이 있다면 로그인 요청이 어떻게 컨트롤러에서 서비스로, 그리고 리포지토리로 전달되어 데이터베이스와 상호작용하는지 흐름을 따라가며 분석합니다.
이러한 작업을 반복하면 개별 기능뿐만 아니라 전체적인 서비스 흐름을 점차 이해할 수 있습니다.</p>
<h4 id="3-도움이-되는-추가-자료-및-팁">3. 도움이 되는 추가 자료 및 팁</h4>
<ul>
<li><strong>ERD(Entity-Relationship Diagram)</strong>를 그려보면 데이터베이스 구조를 이해하는 데 도움이 됩니다.</li>
<li>주석 달기: 코드를 읽으면서 이해한 내용을 주석으로 남기면 나중에 다시 참고하기 좋습니다.</li>
<li>API 문서: 프로젝트에 API 문서나 Swagger가 있다면 이를 활용하여 전체 API 목록과 기능을 확인할 수 있습니다.</li>
</ul>
<blockquote>
<p>위의 내용을 참고해서 하나 하나 분석해보고 그려내면 언젠가는 내 머리속에도 큰 틀이 잡히겠지?</p>
</blockquote>
