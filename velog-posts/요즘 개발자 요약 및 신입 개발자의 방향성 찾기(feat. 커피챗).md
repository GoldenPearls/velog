<h1 id="1장-이런-개발자가-되고-싶어">1장 이런 개발자가 되고 싶어</h1>
<p><img alt="" src="https://velog.velcdn.com/images/prettylee620/post/e6e104c1-fbf8-4f33-b23e-713993d17faa/image.png" /></p>
<blockquote>
<p><strong>우리는 항상 왜 그런가</strong>라는 질문을 던지고 고민해야 한다. 내가 생각하는 개발자의 모습 구체적인 모습이 그려지지 않는다면, 먼저 자주 사용하는 서비스나 회사를 떠올려보자..!</p>
</blockquote>
<h2 id="1-회사가-정의하는-개발자">1. 회사가 정의하는 개발자</h2>
<p>어떤 문제를 어떻게 해결했는지를 보며, 세상에서 정의하는 개발자의 모습이 문제를 효율적으로 해결하는 문제해결사라면 회사가 정의하는 개발자는 <strong>특정 문제를 회사만의 방식으로 탁월하게 해결하는 문제해결사</strong></p>
<h2 id="2-내가-정의하는-개발자">2. 내가 정의하는 개발자</h2>
<p>같은 개발자여도 모두 다른 커리어를 쌓으면서 그 과정에서 <strong>자신만의 장점이 녹아든 길을 직접 만들어야 함</strong></p>
<blockquote>
<p>즉, 어떤 문제를 해결하고 어떤 가치를 만드는 개발자가 되고 싶은지 고민하며 정의</p>
</blockquote>
<p><img alt="" src="https://velog.velcdn.com/images/prettylee620/post/cef83f7f-3198-4f85-bde9-6ee75ba3c30c/image.png" /></p>
<h3 id="1-좋은-개발자의-덕목--잘하고-싶은-것-생각">1) 좋은 개발자의 덕목 + 잘하고 싶은 것 생각</h3>
<p>좋은 개발자의 덕목은 무엇인가를 고민하며 동시에 내가 잘하고 싶은 것이 무엇인지를 생각하는 것이 중요하다고 한다.</p>
<ul>
<li>예시로 문제 정의, 문제 해결, 해결방안 공유라고 생각하고 함께 밝게 나답게 성장할 수 있는 교육환경을 구축하는 개발자 </li>
</ul>
<p>한 번에 정의하는 것이 아닌 <code>꾸준히 재정의</code>의 필요하다고 함</p>
<h3 id="2-내-호기심을-자극하는-것은-무엇인가">2) 내 호기심을 자극하는 것은 무엇인가</h3>
<blockquote>
<p>나와 거리가 먼 개발자보다는 어떤 변화를 일으키고 싶은지 질문하며 접근 </p>
</blockquote>
<p>너무 많다면 자신이 어떤 부분에 가장 많는 돈을 쓰는지 살펴보고 실패한 경험 또한 공유하자.
<strong>회사의 문화와 비전을 이해</strong>하면 해당 회사가 자신에게 적합한지, 그 회사의 개발자는 어떤 종류의 문제를 해결해야 하는지를 알 수 있다. </p>
<p>그렇게 되면 자신의 진로와 정체성을 더 명확하게 정의할 수 있으며 내가 지향하는 가치를 추구하고 내가 풀고 싶은 문제를 해결하는 기업을 찾자.</p>

<p>프로그래밍의 학습의 시작은 피드백과 회고</p>

<p>즉각적인 피드백이 필요하다. 그래야 더 효과적으로 성장할 수 있기 때문에 마치 경험치를 얻고 성장하는 속도를 보며 재미를 느끼는 것처럼 말이지 </p>
<p>주기적인 회고가 필요한 이유는 <strong>피드백을 잘 반영하기 위해서</strong>이다.</p>

<h2 id="4-어디서-배워야-할까">4. 어디서 배워야 할까</h2>
<h3 id="1-📕-책--읽지만-말고-적용하자">1) 📕 책 : 읽지만 말고 적용하자</h3>
<p><img alt="" src="https://velog.velcdn.com/images/prettylee620/post/2399c359-90a1-40ed-a67b-65728f078e9f/image.png" /></p>
<ol>
<li><p>살펴보기

</li>
<li><p>질문하기
: 각 장이나 절을 읽기 전에 스스로 질문하는 과정으로 적극적인 학습 태도</p>
</li>
<li><p>읽기

</ol>
<h3 id="2-👩💻-온라인-강의--강사-생각과-내-생각의-차이를-찾자">2) 👩‍💻 온라인 강의 : 강사 생각과 내 생각의 차이를 찾자</h3>
<p><img alt="" src="https://velog.velcdn.com/images/prettylee620/post/218a3dbb-1abe-4c82-afec-9cec8de4f2cb/image.png" /></p>
<ol>
<li><p>목차보기
: 배울 내용 시뮬레이션 </p>
</li>
<li><p>내가 생각 한 내용과 강사의 이야기 비교
: 왜 같고 다른지 이유를 파악하며 지식을 능동적 흡수</p>
</li>
<li><p>과거의 경험과 강의 내용을 적극적 접목
: 정리하는 것에 그치는 것이 아닌 경험 적극적 반영</p>
</li>
</ol>
<blockquote>
<p>프로그래밍 학습에 있어 하나의 정답은 없다. <strong>최적의 학습 환경은 스스로 만들어 나가야 한다</strong></p>
</blockquote>
<h3 id="3-❓-어떤-것부터-공부해야-할까">3) ❓ 어떤 것부터 공부해야 할까</h3>
<p>프로그래밍의 본질은 문제 해결이므로 어떤 문제를 해결할지 먼저 정의하고 그 문제를 효율적으로 해결하기 위해 적절한 도구 사용하는 것이 중요
=&gt; <code>해결하고 싶은 문제</code>에 초점을 맞춰 학습 방향을 설정</p>
<h4 id="step-1-내가-해결해보고-싶은-문제와-맥락-찾기">Step 1 내가 해결해보고 싶은 문제와 맥락 찾기</h4>
<p>평소 관심을 가졌던 회사나 서비스를 살펴보면서 내가 정의한 개발자의 모습까지 고민한다면 해결하고자 하는 문제를 찾는 과정은 더 쉬워질 것</p>
<h4 id="step-2-실무자에게-연락하기">Step 2 실무자에게 연락하기</h4>
<blockquote>
<p>** 연락할 수 있는 방법**</p>
</blockquote>
<ul>
<li>인프런 커피챗</li>
<li>커피챗 앱</li>
<li>트위터 </li>
<li>링크드인</li>
<li>오픈 카톡방</li>
</ul>
<p>문제를 해결하기 위한 역량을 실무자에게 물어보며 프로그래밍으로 문제를 해결하는 사람을 찾자(링크드인 콜드메일)</p>
<h4 id="step-3-각-팀에서-원하는-역량-조사하기">Step 3 각 팀에서 원하는 역량 조사하기</h4>
<p>원하는 직군과 회사에서 일하기 위해서는 우선순위가 가장 높은 역량 파악하고 이를 기반으로 성장하는 사이클 만들기</p>
<blockquote>
<ul>
<li>팀 문화는 어떤지</li>
</ul>
</blockquote>
<ul>
<li>테스트 코드를 작성하는지</li>
<li>pr 리뷰는 어떻게 진행하는지</li>
<li>현재 팀에서 가장 목말라하는 역량은 무엇인지</li>
</ul>
<blockquote>
<p>채용 공고에 있는 모든 요건을 충족하는 사람은 거의 없다. <strong>채용 공고에서 어떤 것이 더 중요한 지 파악하고 채운 후 나머지 역량은 어떠한 방향으로 점진적으로 성장시키려고 한다는 가능성을 보여주는 것이 중요</strong></p>
</blockquote>
<p>문제 해결에 필요한 역량을 적극적으로 찾아서 학습하는 사이클을 만들어야 빠르게 성장 가능하고 번아웃이 덜 올 수 있다. 제대로 목표에 다가가고 있는가에 대한 질문을 스스로 하기</p>
<h3 id="4-무엇부터-만들어야-할까">4) 무엇부터 만들어야 할까</h3>
<ol>

</ol>

</li>
</ol>

</blockquote>
<h3 id="6-스터내-내에서의-피드백-전략">6) 스터내 내에서의 피드백 전략</h3>

<h3 id="8--효과적인-코드-리뷰-방법">8)  효과적인 코드 리뷰 방법</h3>
<ol>
<li><p>해결하고 싶은 문제의 의도 드러내기
: 내가 먼저 코드를 복기하다가 문제를 해결하는 경우도 있음 복기 후 피드백 요청</p>
</li>
<li><p>코드 리뷰 과정에서 리액션하기<br />: 질문을 똑똑하게 하며, 피드백 적극적 반영</p>
</li>
<li><p>코드 리뷰 후 회고로 되돌아보기
: 내가 다음에 더 잘하고 싶은 부분이나 만족스러운 부분에 살펴보는 과정과 코드 리뷰 후 규칙 만들기</p>
</li>
</ol>
<h4 id="💥-코드-리뷰-시-검토할-항목">💥 코드 리뷰 시 검토할 항목</h4>
<p><strong>1. 이 코드가 어떤 문제를 해결하는가(특히 중요)</strong>
: 놓친 경우의 수는 없는지 혹은 예상 되는 버그는 없는 지 살펴보는 것이 중요</p>
<p><strong>2. 코드 표준을 지켰는지</strong>
코딩 표준은 코드 가독성, 일관성, 유지 보수성, 버그 위험 감소 등 다양한 이유로 중요하게 여겨짐</p>
<p><strong>3. 성능 및 확장성을 확인했는가</strong>
현재 코드가 컴퓨터의 자원을 필요 이상으로 사용하지 않는디, 사용자의 입력 또는 데이터 수가 증가하면서 처리 속도가 저하되거나 멈추는 일이 없는지 등을 판단한다. </p>
<p><strong>4. 보안 측면에서 취약한 부분은 없는가</strong></p>
<h4 id="💥-체크리스트를-만들고-확인">💥 체크리스트를 만들고 확인</h4>
<blockquote>
